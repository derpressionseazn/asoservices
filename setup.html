👁 In a Blink

One-time setup guide — takes about 3 minutes

STEP 1 Create a Web OAuth Client in Google Cloud

Your existing OAuth client is a Desktop app type — browsers require a Web Application client. Here's exactly what to do:

  1. Open Google Cloud Console → Credentials
  2. Click + CREATE CREDENTIALS → OAuth client ID
  3. Application type → Web application
  4. Name it anything, e.g. In a Blink Web
  5. Under Authorised JavaScript origins click + ADD URI and enter:
    http://localhost:3000
  6. Leave Authorised redirect URIs empty (token flow doesn't need it)
  7. Click CREATE
  8. Copy the new Client ID — it ends in .apps.googleusercontent.com
Open Google Cloud Console →

STEP 2 Make sure the Drive API is enabled

The Google Drive API must be enabled on your project:

  1. Go to Drive API Library page
  2. If it shows ENABLE, click it. If it shows MANAGE, you're already good ✅
Check Drive API →

STEP 3 Update the Client ID in the app

Once you have your new Web Application Client ID, open the file:

web\js\drive.js

And update line 7:

const CLIENT_ID = "YOUR_NEW_CLIENT_ID.apps.googleusercontent.com";

The current Client ID is the Desktop app one and will be rejected by the browser.

STEP 4 Run the app

Double-click launch.bat in the web folder — it starts the server and opens your browser automatically.

Or manually:

cd web && node server.js → open http://localhost:3000

CHECK Pre-flight system check

Click the button below to check your browser is ready:
